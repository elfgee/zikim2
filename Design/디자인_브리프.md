# 지킴진단 디자인 브리프 (Design Brief)

## 문서 정보
- 작성일: 2024년 12월 12일
- 작성자: [Gina Chang]
- 버전: 1.0
- 목적: 와이어프레임 설계를 위한 핵심 정보 제공

---

## 1. 프로젝트 개요

### 1.1 서비스명
**지킴진단 (Zikim Diagnosis)**

### 1.2 서비스 설명
전/월세 매물 탐색 시, 사용자가 해당 매물의 계약 안정성에 대한 심층 진단 리포트를 유료로 제공받아 안전한 계약을 돕는 서비스

### 1.3 핵심 가치 제안
계약 전, 잠재적 위험(권리 관계, 보증금 회수 위험 등)을 사전에 파악하고, 필요시 법적으로 보호받을 수 있는 특약을 제공해줌으로서 사용자의 소중한 보증금을 지키고 심리적 안정감을 제공합니다.

---

## 2. 목표 및 목적

### 2.1 비즈니스 목표
- 매물 문의 건수의 30%가 지킴진단 서비스를 활용하도록 함
- 유료 진단 서비스로 수익 모델 확립
- 전세사기 방지 서비스로 브랜드 신뢰도 향상

### 2.2 사용자 목표
- 계약 전 잠재적 위험(권리 관계, 보증금 회수 위험 등)을 사전에 파악
- 소중한 보증금을 보호하고 심리적 안정감 확보
- 안전한 계약을 위한 객관적이고 신뢰할 수 있는 정보 제공

### 2.3 디자인 목표
- 신뢰감을 주는 전문적인 인터페이스
- 복잡한 정보를 이해하기 쉽게 시각화
- 안전성과 신뢰성을 강조하는 디자인
- 직관적이고 사용하기 쉬운 인터페이스

## 3. 타겟 사용자 및 페르소나

### 3.1 주요 타겟 사용자
1. **전세/월세 계약 예정자**: 안전한 계약을 위해 사전에 위험을 파악하고 싶은 사용자
2. **보증금 규모가 큰 계약자**: 큰 금액의 보증금을 보호하고 싶은 사용자
3. **전세사기 우려가 높은 사용자**: 사회적 이슈로 인한 불안감을 해소하고 싶은 사용자

### 3.2 대표 페르소나

**김지현 (30대 후반, IT 기획자)**
- 전세사기 방지를 최우선으로 고려하는 독신 거주자
- 전세사기에 대한 매우 높은 불안감
- 매물의 안전성(근저당, 전세가율, 집주인 신뢰도) 및 법적 보호 정보를 적극적으로 찾음
- 객관적이고 신뢰할 수 있는 정보 제공을 선호

**김미래 (28세, 신중한 1인 가구)**
- 매물 상세 페이지에서 서비스를 즉시 발견하고 시작할 수 있어야 함
- 결제 직후 리포트를 확인하고, 복잡한 정보를 쉽게 이해할 수 있어야 함

**박지훈 (35세, 안전 우선 주거 이전자)**
- 최소한의 정보만 입력하고 신속하게 결제를 완료할 수 있어야 함
- 며칠 후 다시 리포트를 확인하고 싶을 때 쉽게 접근할 수 있어야 함

### 3.3 사용자 특성 및 니즈
- 전세사기에 대한 매우 높은 불안감
- 매물의 안전성(근저당, 전세가율, 집주인 신뢰도) 및 법적 보호 정보를 적극적으로 찾음
- 객관적이고 신뢰할 수 있는 정보 제공을 선호
- 복잡한 법률 용어를 이해하기 어려움
- 빠르고 간편한 서비스 이용을 선호

## 4. 주요 기능 및 화면

### 4.1 핵심 기능 (우선순위 순)

#### 1. 매물 진단 종합 리포트 생성 (Must Have)
- 권리관계 분석 및 계약 안정성 진단을 포함한 종합 리포트
- 진단 항목:
  - 매물진단: 권리 관계, 근저당, 전세가율 등 종합 분석
  - 소유자(집주인) 안정성 확인: 토지/건물 소유자 일치 여부, 임대사업자 여부
  - 시세 진단: 시세 대비 전세금 비율 평가, 보증금 회수 위험도 평가
  - 대출/보험 진단: 보증보험 가입 가능 여부 확인
  - 범죄/치안: CCTV 등 보안 시설 분석, 범죄발생율, 동네 유흥업소 비교
  - 생활/편의: 편의시설 분포 지도, 정주여건 비교

#### 2. 법적 보호 특약 제공 (Must Have)
- 진단 결과에 따라 필요시 법적으로 보호받을 수 있는 특약 제공
- 종합 리포트에 포함
- 특약 조항을 클릭하여 copy&paste 가능

#### 3. 유료 결제 프로세스 (Must Have)
- 토스 브랜드 페이 API를 활용한 간결하고 신속한 결제
- 리포트 발급 건당 금액 청구

#### 4. 리포트 다운로드 및 저장 (Should Have)
- PDF 다운로드 기능
- 리포트 보관 및 재확인

#### 5. 진단 이력 관리 (Nice to Have)
- 'My 직방' 섹션 내 '나의 지킴 진단 리포트' 메뉴
- 리포트 목록 조회 및 재확인

### 4.2 주요 화면 목록

1. **서비스 발견 화면**
   - 매물 상세 페이지 / 게이트웨이(서비스홈) / My직방 배너
   - CTA: "보증금 안전 진단 받기"

2. **진단 시작 화면**
   - 매물 정보 입력 (주소/면적, 보증금, 월세(선택), 희망 계약 기간)
   - 직방 매물 정보 자동 완성

3. **결제 화면**
   - 진단 리포트 가격 확인
   - 토스 브랜드 페이 결제 진행

4. **리포트 생성 대기 화면**
   - 진행 상황 표시
   - 예상 소요 시간 안내 (최대 20분)

5. **리포트 상세 화면 (종합 리포트)**
   - 종합 안전 등급 요약 (상단 강조)
   - 진단 항목별 상세 정보
   - 법률 용어 툴팁
   - 특약 조항 (위험 요소 발견 시)
   - PDF 다운로드 버튼

6. **나의 지킴 진단 리포트 (My 직방)**
   - 리포트 목록 (최신순 정렬)
   - 리포트 상세 확인

## 5. 사용자 플로우

### 5.1 전체 플로우

```
서비스 발견 (매물 상세/게이트웨이/My직방)
  ↓
보증금 안전 진단 받기 버튼 클릭
  ↓
진단 시작 화면 (정보 입력)
  ↓
결제 화면 (토스 브랜드 페이)
  ↓
리포트 생성 대기 (최대 20분)
  ↓
푸시 알림 발송
  ↓
리포트 상세 화면 확인
  ↓
PDF 다운로드 또는 My 직방에서 재확인
```

#### 시나리오 1: 서비스 발견 및 진입

- **행동**: 매물 상세 페이지에서 '지킴 진단 서비스' 즉시 발견
- **요구사항**: 눈에 띄는 배너, 명확한 CTA

#### 시나리오 2: 간결한 정보 입력 및 결제
- **사용자**: 박지훈(35세)
- **행동**: 최소한의 정보만 입력하고 신속하게 결제 완료
- **요구사항**: 자동 완성, 최소 입력, 간결한 결제 UI

#### 시나리오 3: 리포트 확인
- **사용자**: 김미래(28세)
- **행동**: 리포트 내용(특히 위험도)을 쉽게 확인하고 PDF 다운로드
- **요구사항**: 종합 안전 등급 상단 강조, 법률 용어 툴팁, PDF 다운로드

## 6. 디자인 요구사항

### 6.1 디자인 시스템
- **ZUIX 2.0 디자인 시스템** 준수
- 디자인 시스템 링크: [ZUIX 2.0](https://www.figma.com/design/4ZDxigMgcTwzJmNlrmY0iq/ZUIX-2.0?m=auto&t=YCrqYD4ySFnpuv4W-6)
- 상세 적용 방안: `Design/specifications/ZUIX2.0_적용방안.md` 참조

### 6.2 색상
- **Primary Color**: ZUIX 2.0 Green (안전성과 신뢰성 강조)
- **Status Colors**: 위험도 표시를 위한 green/yellow/red 색상 코딩
- **Color Style**: green

### 6.3 타이포그래피
- ZUIX 2.0 타이포그래피 시스템 준수
- 가독성이 높은 폰트 사용
- 리포트 내 법률 용어는 툴팁으로 설명 제공
- 종합 안전 등급은 큰 폰트 사이즈로 강조

### 6.4 스페이싱
- 정보의 계층 구조를 명확히 하는 스페이싱
- ZUIX 2.0 스페이싱 시스템 활용

### 6.5 컴포넌트 활용
**기존 ZUIX 2.0 컴포넌트:**
- Button (CTA, 결제, 다운로드)
- Input (정보 입력)
- Card (리포트 목록, 리포트 섹션)
- Badge/Label (안전 등급, 위험도 레벨)
- Progress/Loading (리포트 생성 대기)
- Modal/Dialog (결제 팝업)
- Tooltip (법률 용어 설명)

**새로 제작 필요한 컴포넌트:**
- 종합 안전 등급 요약 카드
- 진단 항목별 상세 정보 카드
- 특약 조항 카드
- 위험도 레벨 시각화 컴포넌트 (메타포)

### 6.6 레이아웃
- 모바일 우선 설계 (Mobile First)
- 반응형 레이아웃 고려
- 리포트 상세 화면은 스크롤 가능한 긴 형태로 설계

## 7. 화면별 디자인 요구사항

### 7.1 서비스 발견 화면
**요구사항:**
- 매물 상세 페이지 상단 또는 게이트웨이(서비스홈)와 My직방 페이지에서 눈에 띄는 배너 형태
- CTA: "보증금 안전 진단 받기"
- ZUIX 2.0 Green 색상 활용
- 클릭 시 진단 시작 화면으로 바로 전환

### 7.2 진단 시작 화면
**요구사항:**
- 최소한의 정보만 입력받도록 간결한 레이아웃
- 매물 페이지에서 시작한 경우: 주소/면적 자동 완성
- 메뉴에서 시작한 경우: 주소/면적 직접 입력 가능
- 입력 항목: 보증금, 월세(선택), 희망 계약 기간 (3가지 이내)
- 입력 완료 시 결제 화면으로 이동

### 7.3 결제 화면
**요구사항:**
- 진단 리포트 가격 명확히 표시
- 토스 브랜드 페이 API 팝업 안정적 호출
- 3단계 이내의 터치/클릭으로 결제 완료
- 결제 실패 시 명확한 오류 메시지 및 재시도 버튼

### 7.4 리포트 생성 대기 화면
**요구사항:**
- 진행 상황 표시 (Progress 컴포넌트)
- 예상 소요 시간 안내 (최대 20분)
- 사용자가 다른 작업을 할 수 있음을 안내
- 리포트 생성 완료 시 푸시 알림 발송

### 7.5 리포트 상세 화면 (종합 리포트)
**요구사항:**
- **종합 안전 등급 요약**이 가장 상단에 크게 강조 (시각적 메타포 포함)
- 진단 항목별 상세 정보 카드 형태로 표시:
  - 매물진단
  - 소유자(집주인) 안정성
  - 시세 진단
  - 대출/보험 진단
  - 범죄/치안
  - 생활/편의
- 복잡한 법률 용어는 툴팁으로 설명 제공
- 위험 요소 발견 시 특약 조항 섹션 포함
- 특약 조항은 클릭하여 copy&paste 가능
- 리포트 상단에 PDF 다운로드 버튼
- 스크롤 가능한 긴 형태의 레이아웃

### 7.6 나의 지킴 진단 리포트 (My 직방)
**요구사항:**
- 'My 직방' 메뉴 내 '나의 지킴 진단 리포트' 메뉴 신설
- 리포트 목록 최신순 정렬
- 각 리포트 항목: 진단 일자, 매물 주소 표시
- 리포트 클릭 시 리포트 상세 화면으로 이동
- 진단 매물이 삭제되어도 리포트 영구 접근 가능

## 9. 제약사항 및 고려사항

### 9.1 기술적 제약
- 토스 브랜드 페이 API 연동 필요
- 리포트 생성 시간: 최대 20분 소요
- 푸시 알림 및 카카오톡/SMS 알림 필요

### 9.2 비즈니스 제약
- 유료 서비스 (건당 결제)
- 리포트 생성 전 결제 완료 필요
- 환불 정책 준수

### 9.3 사용자 경험 제약
- 리포트 생성 시간이 길어 사용자 이탈 가능
- 복잡한 정보를 이해하기 쉽게 시각화 필요
- 법률 용어를 일반 사용자가 이해할 수 있도록 설명 필요

### 9.4 디자인 제약
- ZUIX 2.0 디자인 시스템 준수 필수
- 직방 앱 내 다른 서비스와 일관성 유지
- 모바일 우선 설계

## 12. 와이어프레임 설계 가이드

### 12.1 설계 우선순위
1. **서비스 발견 화면** (배너 디자인)
2. **진단 시작 화면** (정보 입력 폼)
3. **결제 화면** (결제 프로세스)
4. **리포트 생성 대기 화면** (로딩 상태)
5. **리포트 상세 화면** (종합 리포트 - 가장 복잡)
6. **나의 지킴 진단 리포트** (리포트 목록)

### 12.2 설계 시 중점 사항
- **신뢰감**: 전문적이고 신뢰할 수 있는 디자인
- **명확성**: 복잡한 정보를 이해하기 쉽게 시각화
- **간결성**: 최소한의 정보만 표시, 불필요한 요소 제거
- **일관성**: ZUIX 2.0 디자인 시스템 준수
- **접근성**: 모든 사용자가 쉽게 사용할 수 있도록

### 12.3 특별 고려사항
- 리포트 상세 화면의 종합 안전 등급 요약은 시각적 메타포를 활용하여 직관적으로 표현
- 법률 용어는 툴팁으로 설명 제공하여 사용자가 이해하기 쉽도록
- 특약 조항은 텍스트 형태로 제공하여 copy&paste 가능하도록
- 리포트 생성 대기 시간이 길므로 사용자가 다른 작업을 할 수 있음을 명확히 안내
