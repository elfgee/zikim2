# 지킴진단 와이어프레임 리뷰 가이드

## 목적
UXPilot으로 생성한 와이어프레임을 PRD 및 디자인 브리프와 대조하여 요구사항 충족 여부를 확인하고, 개선 사항을 도출합니다.

## 리뷰 체크리스트

### 1. 서비스 발견 화면 (6-서비스 발견 화면.html)

#### PRD 요구사항 충족 여부
- [ ] 매물 상세 페이지 상단 또는 게이트웨이(서비스홈)와 My직방 페이지에서 눈에 띄는 배너 형태로 노출
- [ ] 배너에 '보증금 안전 진단 받기'와 같은 명확한 CTA 문구 포함
- [ ] CTA 클릭 시 서비스 소개 및 정보 입력 화면으로 바로 전환

#### 디자인 브리프 요구사항 충족 여부
- [ ] ZUIX 2.0 Green 색상 활용
- [ ] 눈에 띄는 배너 디자인
- [ ] 명확한 CTA 버튼

#### 개선 사항
- [ ]

### 2. 진단 시작 화면 (1-헤더 영역에 컨텐츠 영역 잘림 해결.html, 2-한 섹션 위로 이동.html)

#### PRD 요구사항 충족 여부
- [ ] 매물 페이지에서 직접 진단을 시작한 경우, 주소/면적이 자동으로 완성되어 사용자 입력이 필요 없음
- [ ] 매물 페이지가 아닌 메뉴에서도 직접 주소/면적을 입력하여 진단을 시작할 수 있음
- [ ] 사용자가 입력해야 하는 정보는 '보증금', '월세(선택)', '희망 계약 기간' 3가지 이내로 최소화
- [ ] 입력 완료 시 결제 화면으로 이동

#### 디자인 브리프 요구사항 충족 여부
- [ ] 최소한의 정보만 입력받도록 간결한 레이아웃
- [ ] 자동 완성 기능 표시
- [ ] ZUIX 2.0 Input 컴포넌트 활용

#### 개선 사항
- [ ] 

---

### 3. 리포트 생성 대기 화면 (3-리포트 생성 대기 화면.html)

#### PRD 요구사항 충족 여부
- [ ] 진행 상황 표시
- [ ] 예상 소요 시간 안내 (최대 20분)
- [ ] 리포트 생성이 완료되면 푸시 알림 발송
- [ ] 사용자는 리포트 생성 중에도 다른 작업을 할 수 있음

#### 디자인 브리프 요구사항 충족 여부
- [ ] ZUIX 2.0 Progress/Loading 컴포넌트 활용
- [ ] 진행 상황을 명확히 표시
- [ ] 예상 소요 시간 안내

#### 개선 사항
- [ ]

### 4. 리포트 상세 화면 (4-리포트 상세 화면.html)

#### PRD 요구사항 충족 여부
- [ ] 종합 안전 등급 요약이 가장 상단에 크게 강조되어 표시
- [ ] 진단 항목별 상세 정보 표시:
  - [ ] 매물진단
  - [ ] 소유자(집주인) 안정성
  - [ ] 시세 진단
  - [ ] 대출/보험 진단
  - [ ] 범죄/치안
  - [ ] 생활/편의
- [ ] 복잡한 법률 용어는 툴팁으로 설명 제공
- [ ] 위험 요소 발견 시 특약 조항 섹션 포함
- [ ] 특약 조항을 클릭하여 copy&paste 가능
- [ ] 리포트 상단에 PDF 다운로드 버튼 제공
- [ ] 스크롤 가능한 긴 형태의 레이아웃

#### 디자인 브리프 요구사항 충족 여부
- [ ] 종합 안전 등급 요약이 시각적 메타포를 활용하여 직관적으로 표현
- [ ] 진단 항목별 상세 정보 카드 형태로 표시
- [ ] 법률 용어 툴팁 제공
- [ ] 특약 조항 섹션 포함
- [ ] PDF 다운로드 버튼 명확히 표시

#### 개선 사항
- [ ]

- [ ] 리포트 목록 최신순 정렬
- [ ] 진단 일자 및 매물 주소 표시

## 전체 플로우 검증

### 화면 간 전환
- [ ] 서비스 발견 → 진단 시작 화면 전환
- [ ] 진단 시작 → 결제 화면 전환
- [ ] 결제 → 리포트 생성 대기 화면 전환
- [ ] 리포트 생성 대기 → 리포트 상세 화면 전환
- [ ] 리포트 상세 → 나의 지킴 진단 리포트 전환
