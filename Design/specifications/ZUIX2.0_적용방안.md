## 1. ZUIX 2.0 디자인 시스템 개요

### 1.1 ZUIX 2.0이란?

ZUIX 2.0은 직방에서 운영하는 서비스들의 UI 컴포넌트를 제작, 운영 및 관리하는 디자인 시스템입니다.

**핵심 특징:**
- **One Code, Multi Use**: 하나의 코드베이스로 웹과 모바일 앱에서 일관된 UX 제공
- **기술 스택**: React, React Native, React Native for Web 기반
- **컴포넌트 재사용성**: 표준화된 컴포넌트로 개발 효율성 향상
- **디자인 협업**: Zeplin 등 협업 툴을 통한 디자인-개발 협업 프로세스

**디자인 시스템 링크:**
- [ZUIX 2.0 Figma](https://www.figma.com/design/4ZDxigMgcTwzJmNlrmY0iq/ZUIX-2.0?m=auto&t=YCrqYD4ySFnpuv4W-6)

## 2. 지킴진단 서비스 디자인 요구사항 분석

### 2.1 PRD 기반 디자인 목표

**디자인 목표:**
- 신뢰감을 주는 전문적인 인터페이스
- 복잡한 정보를 이해하기 쉽게 시각화
- 안전성과 신뢰성을 강조하는 디자인
- 직관적이고 사용하기 쉬운 인터페이스

### 2.2 디자인 요소 요구사항

**색상:**
- 안전성과 신뢰성을 나타내는 색상 팔레트 (color style : green)
- ZUIX 2.0의 green color style 활용

**타이포그래피:**
- 가독성이 높은 폰트 사용
- ZUIX 2.0의 타이포그래피 시스템 준수

**간격:**
- 정보의 계층 구조를 명확히 하는 스페이싱
- ZUIX 2.0의 스페이싱 시스템 활용

### 2.3 주요 화면별 디자인 요구사항

1. **서비스 발견 화면**
   - 매물 상세 페이지 / 게이트웨이 / My직방에서 눈에 띄는 배너
   - CTA: "보증금 안전 진단 받기"

2. **진단 시작 화면**
   - 최소한의 정보 입력 (보증금, 월세(선택), 희망 계약 기간)
   - 직방 매물 정보 자동 완성

3. **결제 화면**
   - 토스 브랜드 페이 API 연동
   - 간결하고 신속한 결제 UI/UX

4. **리포트 생성 대기 화면**
   - 진행 상황 표시
   - 예상 소요 시간 안내

5. **리포트 상세 화면**
   - 종합 안전 등급 요약이 상단에 크게 강조
   - 복잡한 법률 용어 툴팁 제공
   - PDF 다운로드 기능
   - 특약 조항 포함 (위험 요소 발견 시)

6. **나의 지킴 진단 리포트 (My 직방)**
   - 리포트 목록 (최신순 정렬)
   - 리포트 상세 확인

## 3. ZUIX 2.0 적용 방안

### 3.1 디자인 시스템 통합 원칙

**일관성 유지:**
- 지킴진단 서비스의 모든 UI 컴포넌트는 ZUIX 2.0의 표준에 맞게 설계
- 직방 앱 내 다른 서비스와 일관된 디자인 언어 유지
- 사용자가 직방 앱 내에서 자연스러운 경험을 할 수 있도록 함

**컴포넌트 재사용:**
- ZUIX 2.0에서 제공하는 기존 컴포넌트 우선 활용
- 필요시 ZUIX 2.0 컴포넌트를 커스터마이징하여 사용
- 새로운 컴포넌트는 ZUIX 2.0 디자인 시스템 가이드라인을 준수하여 제작

### 3.2 색상 시스템 적용

**Green Color Style 활용:**
- ZUIX 2.0의 green color style을 지킴진단 서비스의 주요 색상으로 사용
- 안전성과 신뢰성을 나타내는 green 계열 색상 팔레트 적용
- 위험도 표시 시 green (안전) / yellow (주의) / red (위험) 등 색상 코딩 활용
- 결제하기 등의 CTA는 ZUIX 2.0의 primary color 를 그대로 유지
- 지킴 brand color 는 위험도에서 긍정적인 상태와, 기능 소개에 활용

**색상 사용 가이드:**
- Primary Color: ZUIX 2.0 Green (안전성 강조)
- Secondary Color: ZUIX 2.0의 보조 색상 활용
- Status Colors: 위험도 표시를 위한 상태 색상 (green/yellow/red)

### 3.3 타이포그래피 시스템 적용

**ZUIX 2.0 타이포그래피 준수:**
- ZUIX 2.0에서 정의한 폰트 시스템 사용
- 리포트 상세 화면의 복잡한 정보를 가독성 높게 표시
- 계층 구조를 명확히 하는 타이포그래피 스케일 활용

**특별 고려사항:**
- 리포트 내 법률 용어는 툴팁으로 설명 제공
- 종합 안전 등급은 큰 폰트 사이즈로 강조
- 본문 텍스트는 가독성을 최우선으로 고려

### 3.4 컴포넌트 활용 방안

**기존 ZUIX 2.0 컴포넌트 활용:**

1. **Button 컴포넌트**
   - CTA 버튼: "보증금 안전 진단 받기"
   - 결제 버튼
   - PDF 다운로드 버튼
   - 특약 조항 복사 버튼

2. **Input 컴포넌트**
   - 진단 시작 화면의 정보 입력 필드
   - 보증금, 월세, 계약 기간 입력

3. **Card 컴포넌트**
   - 리포트 목록 아이템
   - 리포트 상세 화면의 섹션 카드

4. **Badge/Label 컴포넌트**
   - 종합 안전 등급 표시
   - 위험도 레벨 표시

5. **Progress/Loading 컴포넌트**
   - 리포트 생성 대기 화면
   - 진행 상황 표시

6. **Modal/Dialog 컴포넌트**
   - 결제 팝업
   - 특약 조항 설명 모달

7. **Tooltip 컴포넌트**
   - 법률 용어 설명
   - 복잡한 정보 설명

**새로 제작이 필요한 컴포넌트:**

1. **리포트 상세 화면 전용 컴포넌트**
   - 종합 안전 등급 요약 카드
   - 진단 항목별 상세 정보 카드
   - 특약 조항 카드
   - PDF 다운로드 버튼

2. **진단 항목 시각화 컴포넌트**
   - 위험도 레벨 표시 (시각적 메타포)
   - 차트/그래프 (시세 진단, 범죄/치안 데이터 등)
   - 지도 컴포넌트 (편의시설 분포)

### 3.5 스페이싱 및 레이아웃

**ZUIX 2.0 스페이싱 시스템 활용:**
- 정보의 계층 구조를 명확히 하는 스페이싱
- 리포트 상세 화면의 복잡한 정보를 체계적으로 배치
- 일관된 간격으로 시각적 안정성 확보

**레이아웃 가이드:**
- 모바일 우선 설계 (Mobile First)
- 반응형 레이아웃 고려
- 리포트 상세 화면은 스크롤 가능한 긴 형태로 설계

### 3.6 아이콘 및 일러스트레이션

**ZUIX 2.0 아이콘 시스템 활용:**
- 표준 아이콘은 ZUIX 2.0 아이콘 라이브러리에서 활용
- 지킴진단 서비스 전용 아이콘은 ZUIX 2.0 스타일 가이드 준수

**일러스트레이션:**
- 안전성과 신뢰성을 나타내는 일러스트레이션
- 리포트 상세 화면의 시각적 요소

## 4. 화면별 ZUIX 2.0 적용 상세

### 4.1 서비스 발견 화면

**적용 컴포넌트:**
- Card 컴포넌트 (배너 형태)
- Button 컴포넌트 (CTA)

**디자인 요구사항:**
- ZUIX 2.0의 green color style 활용
- 눈에 띄는 배너 디자인
- CTA 버튼 명확히 표시

### 4.2 진단 시작 화면

**적용 컴포넌트:**
- Input 컴포넌트 (정보 입력)
- Button 컴포넌트 (다음 단계)
- Form 컴포넌트 (입력 폼)

**디자인 요구사항:**
- 최소한의 정보만 입력받도록 간결한 레이아웃
- 자동 완성 기능을 위한 Input 컴포넌트 활용
- ZUIX 2.0의 Form 가이드라인 준수

### 4.3 결제 화면

**적용 컴포넌트:**
- Modal/Dialog 컴포넌트 (토스 브랜드 페이 팝업)
- Button 컴포넌트 (결제 버튼)

**디자인 요구사항:**
- 간결하고 신속한 결제 UI/UX
- 토스 브랜드 페이 API 연동에 맞는 레이아웃
- 결제 정보 안내 텍스트

### 4.4 리포트 생성 대기 화면

**적용 컴포넌트:**
- Progress/Loading 컴포넌트
- Card 컴포넌트 (대기 상태 표시)

**디자인 요구사항:**
- 진행 상황을 명확히 표시
- 예상 소요 시간 안내
- 사용자가 다른 작업을 할 수 있음을 안내

### 4.5 리포트 상세 화면

**적용 컴포넌트:**
- Card 컴포넌트 (각 진단 항목)
- Badge/Label 컴포넌트 (안전 등급)
- Tooltip 컴포넌트 (법률 용어 설명)
- Button 컴포넌트 (PDF 다운로드)
- 새로 제작: 종합 안전 등급 요약 카드
- 새로 제작: 진단 항목별 상세 정보 카드
- 새로 제작: 특약 조항 카드

**디자인 요구사항:**
- 종합 안전 등급 요약이 상단에 크게 강조
- 복잡한 정보를 이해하기 쉽게 시각화
- 법률 용어 툴팁 제공
- PDF 다운로드 버튼 명확히 표시
- 특약 조항은 복사 가능하도록 텍스트 형태로 제공

### 4.6 나의 지킴 진단 리포트 (My 직방)

**적용 컴포넌트:**
- List 컴포넌트 (리포트 목록)
- Card 컴포넌트 (리포트 아이템)
- Navigation 컴포넌트 (My 직방 메뉴)

**디자인 요구사항:**
- 리포트 목록 최신순 정렬
- 진단 일자 및 매물 주소 표시
- 리포트 클릭 시 상세 화면으로 이동

## 5. 커스터마이징 가이드라인

### 5.1 ZUIX 2.0 컴포넌트 커스터마이징

**원칙:**
- ZUIX 2.0의 기본 컴포넌트를 최대한 활용
- 필요한 경우에만 커스터마이징
- 커스터마이징 시에도 ZUIX 2.0의 디자인 원칙 준수

**커스터마이징 예시:**
- 리포트 상세 화면의 종합 안전 등급 카드: ZUIX 2.0 Card 컴포넌트를 기반으로 커스터마이징
- 위험도 레벨 표시: ZUIX 2.0 Badge 컴포넌트를 기반으로 시각적 메타포 추가

### 5.2 새로운 컴포넌트 제작

**제작 원칙:**
- ZUIX 2.0의 디자인 토큰 (색상, 타이포그래피, 스페이싱) 활용
- ZUIX 2.0의 컴포넌트 구조 패턴 준수
- 다른 직방 서비스와 일관성 유지

**제작이 필요한 컴포넌트:**
1. 종합 안전 등급 요약 카드
2. 진단 항목별 상세 정보 카드
3. 특약 조항 카드
4. 위험도 레벨 시각화 컴포넌트

## 8. 적용 일정

### 8.1 단계별 적용 계획

**Phase 1: 디자인 시스템 리뷰 및 준비**
- ZUIX 2.0 디자인 시스템 상세 리뷰
- 지킴진단 서비스 적용 방안 확정
- 필요한 컴포넌트 식별

**Phase 2: 와이어프레임 작성**
- ZUIX 2.0 컴포넌트를 활용한 와이어프레임 작성
- 화면 구조 및 레이아웃 설계

**Phase 3: 프로토타입 제작**
- Figma를 활용한 인터랙티브 프로토타입 제작
- 사용자 플로우 검증

**Phase 4: 상세 디자인**
- ZUIX 2.0 디자인 시스템을 적용한 상세 디자인 완성
- 커스터마이징 및 새로운 컴포넌트 제작

**Phase 5: 디자인 스펙 문서 작성**
- 개발팀 전달용 상세 스펙 문서 작성
- 컴포넌트 사용 가이드 제공
